# Agent 能力要求与边界

## 必需能力

| 能力 | Phase | 说明 |
| --- | --- | --- |
| **Shell 执行** | 0, 3–8, 9 | `python3`、`pip`、`node`（B 路线） |
| **文件读写** | 全阶段 | parquet/csv、py 脚本、docx、PDF |
| **网络访问** | 1–2 | Semantic Scholar、NBER/SSRN 等（可降级为仅用户提供文献） |

## 可选能力

| 能力 | 用途 |
| --- | --- |
| PDF 文本提取 | Phase 2 精读；扫描版 PDF 需用户 OCR |
| 图像生成 | 论文插图（非必需） |
| Git | 项目版本管理（建议用户自行 init） |

## 平台差异处理

| 场景 | 处理方式 |
| --- | --- |
| **无 Shell**（部分网页 Agent） | 输出完整 Python 脚本与运行说明，由用户本地执行；根据用户粘贴的结果解读 |
| **无网络** | Phase 1–2 改为用户提供文献列表与 PDF；跳过 API 搜索 |
| **沙箱禁 pip** | 检测失败时给出 `pip install` 命令，请用户在终端执行 |
| **无 Node.js** | Phase 9 强制 A 路线（python-docx） |
| **工作区在 home 而非项目目录** | 先 `mkdir` + 初始化项目结构，或 `move_agent_to_root` 到论文项目 |

## 学术诚信边界

- **不伪造**回归系数、p 值、样本量、文献引用
- **不代写**应署名于用户的原创贡献声明；Skill 定位为辅助工具
- 使用 CSMAR、知网等**付费数据库**时，提醒用户遵守机构许可，Agent 不破解或分发原始数据
- 引用格式遵循 GB/T 7714；`[CIT:]` 占位符须替换为真实文献后再定稿

## 降级路径（最小可行）

用户资源有限时的推荐裁剪：

| 裁剪 | 保留 | 跳过 |
| --- | --- | --- |
| 课程论文 | Phase 0–4 + 9 简版 | 5–8 选做 1–2 项 |
| 仅有数据 | Phase 0, 3–4, 9 | 1–2 用户自备综述 |
| 仅润色结果章 | Phase 5–8 已有表 | 0–4 |

进入裁剪模式前须在《阶段确认单》中写明，避免用户误以为已完成全套实证。

## 阶段完成检查（Agent 自检）

每 Phase 结束前确认：

- [ ] 本 Phase 产出已写入约定路径
- [ ] 用户已口头/书面确认进入下一 Phase
- [ ] `log/` 中无未解释的脚本失败
- [ ] 关键变量已写入上下文（`{{model_framework}}` 等）
